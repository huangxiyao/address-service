Certified Address

match/certifiedAddress

Finds a certified address per a given postal authority. A number of countries have special requirements for the processing of addresses from their countries. An example of such a special processing requirement is the CASS certification of the United States Postal Service (USPS). Operation will fall back to simple validation if it is not supported for a specific country.

"result" Entity
Name Example Description
countryCode "USA" ISO3 country code.
modeUsed "BATCH" The mode that the address engine used when processing the query. Possible values: BATCH, INTERACTIVE, FASTCOMPLETION, CERTIFIED, PARSE, COUNTRYRECOGNITION. For certifiedAddress, this should always say "CERTIFIED".
preferredLanguage "DATABASE" The “preferredLanguage” attribute is used to specify in which language the output should be returned. The default setting for “preferredLanguage” is “DATABASE”. The alphabet in which the data is returned differs from country to country (see Address Doctor reference guide section 5.12.1), but for most countries the output will be Latin, regardless of the selected preferred language. The possible values are DATABASE, ENGLISH, ALTERNATIVE_1, ALTERNATIVE_2, and ALTERNATIVE_3 (The ALTERNATIVE_1/2/3 attributes are reserved for later use, e.g. for multi-language countries like Canada, Belgium or Switzerland. If no alternative is provided as part of the postal reference data, his setting will revert to the default “DATABASE”).
preferredScript "DATABASE" The “preferredScript” attribute is used to specify in which alphabet the output is returned. The default setting for the “preferredScript” attribute is “DATABASE”. The alphabet in which the data is returned differs from country to country. For most countries the output will be Latin I regardless of the selected preferred language. Please see Address Doctor documentation (section 5.12.1). Possible values of this field are: "DATABASE", "POSTAL_ADMIN_PREF", "POSTAL_ADMIN_ALT", "ASCII_SIMPLIFIED", "ASCII_EXTENDED", "LATIN", "LATIN_ALT". In this release, this attribute will always be "DATABASE".
processStatus "V4" The processStatus values describe the result output quality call to AddressDoctor in an overview fashion (for more detailed information on results consult the “elementResultStatus” as well. See Address doctor documentation for a complete explanation of the various status codes. Possible values are: "V4", "V3", "V2", "V1", "C4", "C3", "C2", "C1", "I4", "I3", "I2", "I1", "N1", "N2", "N3", "N4", "N5", "Q3", "Q2", "Q1", "Q0", "S4", "S3", "S2", "S1", "RA", "R9", "R8", "R7", "R6", "R5", "R4", "R3" "R2", "R1", "R0". Please note that when using the certifiedAddress resource it is strictly recommended to only accept records with Vx or Cx status for automated data updates. Ix records need to be reviewed manually before using these results for any data update whatsoever. For a complete interpretation of these values, please see the address doctor documentation.


Individual anonymous "resultData" Entities
Name Comment Example Description
elementInputStatus "00600060600000000060" Provides per element information on the matching of input elements to reference data. Please see Address Doctor documentation for how to interpret this value.
elementResultStatus "88F088E0F000000000E0" Categorizes the result of each standardized attribute in more detail. Please see Address Doctor documentation for how to interpret this value.
elementRelevance "11101010100000000010" In addition to the element status values described previously, information is available on which of the address elements of the address processed are actually relevant from the local postal operator’s point of view. The possible values for each address element are “1” for relevant and “0” otherwise. For any given address, all address elements with a value of “1” must be present for an output address to be deemed valid by the local postal authority. “elementRelevance” may well vary from address to address for countries with different address types, e.g. rural vs. metropolitan addressing. Furthermore, address elements that have actually been validated against reference data (i.e. with elementResultStatus 7 and higher) may override the default elementRelevance value defined for that address element. Please note that “elementRelevance” is really only meaningful for a “ProcessStatus” value of Cx or Vx). Please see Address Doctor documentation for how to interpret this value further.
mailabilityScore "5" AddressDoctor provides an estimate of how likely successful delivery of mail to a corrected address might be, this should be especially useful in determining whether addresses with an Ix process status value should be bothered with for mailing. Possible values are:
5
completely confident
4
almost certain
3
should be fine
2
fair chance
1
risky
0
futile
resultPercentage "82.09" The “ResultPercentage” value gives an indication how similar a result is to the parsed input, values close to 100% imply high similarity. They are mainly provided to allow for filtering out too extensive corrections in records with Cx BATCH “ProcessStatus” value in master data management environments with very stringent data quality requirements. Also, “ResultPercentage” may be used to determine which addressSuggestions results show the least deviation from input. AddressDoctor discourages using “ResultPercentage” values for any other use case scenarios than the two described above.

CASS attributes only partially provided (some databases are missing)
ECA2..4
Reserved for future use
ECA5
CASS attributes provided

ESE1

ESN0
SNA output not available (for this address)
ESN1
SNA attributes provided

EGB0
GB country specific output not available (for this address)
EGB1
GB country specific attributes provided (not necessarily all attributes are populated)
